Frequently Asked Questions

What is the difference between var, let, and const?

In JavaScript, var, let, and const differ in scope, re-declaration, initialization, and hoisting.
Scope: var has function scope, while let and const have block scope.
Re-declaration & Updates: var can be updated and re-declared. let can be updated but not re-declared. const cannot be updated or re-declared.
Initialization: var and let can be declared without initialization, but const must be initialized at declaration.
Hoisting: var is hoisted and initialized with undefined, while let and const are hoisted but not initialized, causing an error if accessed before declaration.
Use let for variables that change and const for constants. Avoid var due to its functional scope and hoisting behavior.

What is the difference between map(), forEach(), and filter()?

map() is used to transform elements in an array and returns a new array with the modified values. It does not change the original array. forEach() is used to iterate over elements in an array and perform operations, but it does not return a new array. It is mainly used for executing side effects, like logging data. filter() creates a new array with elements that meet a specific condition, filtering out the rest.

Explain arrow functions and how they are different from regular functions?

Arrow functions (=>) in JavaScript provide a concise syntax for writing functions. Unlike regular functions, they do not have their own this; instead, they inherit it from the surrounding scope, making them useful in callbacks. They also lack the arguments object and cannot be used as constructors. Arrow functions work best for simple expressions and callbacks, while regular functions are better when this or arguments is needed.

How JavaScript Promises work?

JavaScript Promises handle asynchronous operations, preventing callback hell. A Promise has three states: pending, fulfilled, or rejected. It represents a future value and executes once completed. Use then() to handle success and catch() for errors.

How closures work in JavaScript?

In JavaScript, a closure is a function along with its lexical scope, which allows it to access variables from its outer (enclosing) scope even after that scope has finished executing. A closure allows a function to remember and access variables from the environment in which it was created, even if the function is executed in a different scope.
